系統運作架構說明

這是一個採用本地 GPU 加速引擎的「非同步任務處理」流程，包含了六個主要階段：

1 前端發起請求 (HTML / GitHub Pages)

使用者輸入網址後，JavaScript 將帶有暗號的任務寫入 Firebase 的 tasks 集合（狀態設定為 pending_local），並獲得專屬 ID。

2 雲端中繼站 (Firebase)

擔任前後端溝通的唯一橋樑。它收到了前端貼上來的新任務佈告，默默等待本地後端認領。

3 後端運算引擎 (Local 本機端 GPU)

您本機的 Python 伺服器發現任務後，便會下載音檔 ➔ 調用本地顯卡 GPU 進行 Whisper 轉錄 ➔ 產出摘要。

4 雲端儲存 (Firebase Storage)

運算完成後，將生成的逐字稿 (.srt) 與摘要 (.json) 備份儲存至 Firebase Storage，供日後回頭調閱。

5 結果回傳前端 (Firebase -> HTML)

Firestore 的 即時監聽機制 (onSnapshot) 會立刻捕捉到資料變動，網頁隨即解除轉圈圈。

6 Chat with Gemini (Local Python)

在前端對話視窗中輸入問題後，本地 Python 會透過 Gemini API 進行深入反覆問答。
