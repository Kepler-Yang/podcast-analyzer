系統運作架構說明

這是一個完全採用本地 GPU 加速引擎的「非同步任務處理」流程，包含了五個主要階段：

1 前端發起請求 (HTML / GitHub Pages)

使用者輸入網址後，JavaScript 將帶有暗號的任務寫入 Firebase 的 tasks 集合（狀態設定為 pending_local），防止雲端伺服器搶單。

2 雲端中繼站 (Firebase)

擔任前後端溝通的唯一橋樑。它收到了前端貼上來的新任務佈告，默默等待後端來認領處理。

3 後端運算引擎 (Local 本機端 GPU)

您本機的 Python 伺服器發現 pending_local 任務後，便會無視一切檔案大小限制，透過 yt-dlp 暴力下載，接著調用本地顯示卡的 CUDA 算力給 OpenAI Whisper 轉錄 ➔ 最後交由 Gemini AI 產出高純度分析。

4 雲端儲存 (Firebase Storage)

運算完成後，生成的逐字稿 (.srt) 與結果會被備份儲存至 Firebase Storage 之中，並產生專屬連結供您隨時調閱下載。

5 結果回傳前端 (Firebase -> HTML)

本地腳本將最後的分析結果寫回 Firebase。此時網頁的監聽器立刻捕捉到變動，隨即渲染這份來自本地硬體的心血結晶給您！
